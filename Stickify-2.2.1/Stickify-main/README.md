# Stickify

Stickify 是一个 SwiftUI iOS demo，用于把拍摄或导入的图片制作成贴纸，并在云库、贴纸库和 Playground 中继续整理、预览与组合创作。

当前版本：v2.2 / 2026-06-20

## 功能模块

### Capture

- 使用相机或相册导入图片并生成贴纸。
- 生成贴纸后可加入临时收集云 / Shelf。
- Capture 右侧 Shelf 支持切换云库、添加云库、选择模式、多选删除和原始大小预览。
- 云朵标题使用蓝色，强调当前收集入口。

### Library

- 管理所有云库和贴纸。
- 云库支持重命名和皮肤设置，皮肤会影响云库 UI 颜色，便于快速识别当前库。
- 贴纸支持长按进入选择模式，多选后可批量删除。
- 贴纸命名规则：
  - 自定义名称优先显示，并排在自动编号贴纸前。
  - 未命名贴纸按进入该库的顺序显示为 `贴纸 1`、`贴纸 2`。
  - 清空自定义名称后，该贴纸会重新进入自动编号队列。

### Playground

- Playground 主页包含两个玩法入口：
  - 玩法1：随心贴
  - 玩法2：融合
- 入口卡为 1:1，第一页只显示标题和描述。
- 如果存在最近项目，会在入口页后显示静态叠加画廊式缩略图。
- 玩法页布局：
  - 顶部右侧为草稿箱和保存。
  - 中间为大面积编辑区域。
  - 下方一行为删除、工具、固化。
  - 再下方为贴纸库。
  - 底部保留主 tab 导航位置。
- 贴纸库云库 UI 与 Library 保持一致：云形矢量图、名称和贴纸计数，无卡片底。
- 贴纸按名称递增从左到右排列，计数格式为 `已显示/该云库总数`。
- 工具栏默认收起，支持：
  - 编辑区域比例
  - 网点 / 网格
  - 轴对称
  - 自动对齐 / 吸附
  - 背景图片
  - 添加图片入口
  - 画笔工具
- 玩法1包含堆叠和散射模式。
- 玩法2包含画布裁剪模式，进入玩法2时默认关闭裁剪。
- 草稿箱保存最近项目，上限 10 个：
  - 最近保存项目 5 个
  - 最近未保存项目 5 个

## 项目结构

```text
Stickify/
  Components/        公共 UI 组件，例如云形、底部 tab、贴纸 chip
  Design/            主题颜色和视觉常量
  Features/
    Capture/         拍摄、导入和 ShelfSheet
    Library/         云库和贴纸库管理
    LiveCamera/      相机预览
    Playground/      玩法主页和编辑器
  Models/            demo 状态、数据模型和持久化
  Resources/         AppIcon、LaunchScreen 等资源
  Services/          相机、分割、贴纸渲染和风格渲染服务
StickifyTests/       单元测试
project.yml          XcodeGen 项目配置
Stickify.xcodeproj/  Xcode 工程
```

## 运行环境

- iOS 17.0+
- SwiftUI
- Swift 5
- Xcode 15 或更新版本

## 打开项目

直接用 Xcode 打开：

```text
Stickify.xcodeproj
```

如果需要根据 `project.yml` 重新生成工程，可在安装 XcodeGen 后运行：

```bash
xcodegen generate
```

## 最近更新

### v2.2 / 2026-06-20

- Capture 云朵标题改为蓝色。
- 贴纸库继承选择模式，支持长按、多选、绿色描边、取消和删除确认。
- 云库新增皮肤属性，并合并到库设置中的“库属性”。
- 删除类型、颜色、分类筛选相关功能。
- ShelfSheet 支持云库左右切换、滑动切换、添加云库和原始大小预览。
- Playground 主页重构为玩法入口。
- Playground 玩法页重排为保存 / 草稿箱、编辑区、工具行、贴纸库、tab 的结构。
- 玩法1/玩法2 的贴纸库云库 UI 与 Library 统一。
- 玩法2 裁切模式改为画布裁剪，并在进入玩法2时默认关闭。
- 工具栏默认收起，并增加网格、轴对称、自动吸附开关。

## 备注

当前项目仍是 demo 状态，部分 Playground 工具为交互原型或占位入口，后续可继续补齐真实渲染、项目恢复、历史撤销和导出链路。
